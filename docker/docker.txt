PROCESS :
a). Cloud native buildPacks
b). spring boot maven plugin        -> by springs inbuild maven dependencies it will be created without even explicit creation of docker image
c). layering      -> It is a caching technique where when we build multiple dockerImages It will bee cached and next time we build again it taken less time to build.
d). paketo buildPacks    -> paketo build packs is a customized build that gives jvm and all the required files to compile spring boot application into dockerImage
e). result     -> result is docker image




CONTAINERIZING OUT SPRING BOOT APPLICATION : 
-> USING SPRING BOOT MAVEN DEPENDENCY TO CONTAINERIZING DOCKER APPLICATION : 
./mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=<IMAGE-NAME>
or
mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=ramrandhi/jobappimage  (For Windows)

=> ./mvnw -> maven wrapper
=> spring-boot:build-image -> maven goal that is provided by spring boot maven to build maven dependencies using cloud native build packs
=> -Dspring-boot.build-image.imageName=  -> java system property that is being passed to spring boot image goal.
=> -D here specifies the system property.


-> mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=ramrandhi/jobappimage
-> docker push ramrandhi/jobappimage
// push docker image to repository
-> docker pull ramrandhi/jobappimage
// pull docker image from repository
-> docker run -it -d -p <host-port>:<container-port> --name <name> <image>
// -it -> interactive, -d -> detached mode (for running as a seperate process even if terminal closed docker runs), -p -> port mapping (map the port of container to that of host machine we can use)
-> docker stop <container-id/container-name>
// stop docker using container id or container name
-> docker start <container-id/container-name>
// start docker using container id or container name
-> docker rm <container-id/container-name>
// remove a stopped container using container id or container name
-> docker rmi <image-id/image-name>
// remove a docker image using image id or image name
-> docker ps  
// gives all the running containers
-> docker ps -a 
// shows all the containers as well as running
-> docker images
// to list all the images on the host
-> docker exec -it <container-name/container-id> bash
// if we want to login to a running container the we can use this command
-> docker build -t <username/image> .
// used to build docker file which is already available in repository
-> docker logs <container-name/container-id>
// if we want to see the logs from the running container
docker inspect <container-name/container-id>
-> gives detailed information for a particular container


DOCKER COMPOSE : This will be used to manage all the containers required for us and if we want can run all containers with one single command.






docker run -d --name db -e POSTGRE_PASSWORD=admin@123 postgres
docker run -d --name pgadmin -e PGADMIN_DEFAULT_EMAIL=admin@admin.com -e PGADMIN_DEFAULT_PASSWORD=admin@123 dpage/pgadmin4
docker rm -f db pgadmin
docker network create my-network
docker run -d --name db --network my-network -e POSTGRES_PASSWORD=mysecretpassword postgres
docker run -d --name pgadmin --network my-network -e PGADMIN_DEFAULT_EMAIL=user@domain.com -e PGADMIN_DEFAULT_PASSWORD=SuperSecret dpage/pgadmin4
docker exec -it pgadmin ping db
