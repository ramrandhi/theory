🌳 Tree vs. Binary Tree vs. Binary Search Tree (BST)
  1. Tree (General Tree)
  =>Each node can have any number of children (not limited to two).
  =>No specific rules about where data goes.
  =>Example use: file systems, DOM in HTML, org charts.

       A
     / | \
    B  C  D
      / \
     E   F

  2. Binary Tree
=>Each node has at most 2 children: left and right.
=>No requirement on the values of nodes.
=>Can store data in any order.
=>Left child can be smaller or larger than the parent — no rules enforced.

       10
      /  \
     5    20
    / \
   3   7

  3. Binary Search Tree (BST)
  => A binary tree with a special rule:
    => Left child < parent < Right child
    => This rule must hold for every node in the tree.
  => Because of this rule, search, insert, and delete operations are efficient (O(log n) on average if balanced).

       10
      /  \
     5    15
    / \     \
   2   7     20

| Feature           | Tree           | Binary Tree        | Binary Search Tree (BST)           |
| ----------------- | -------------- | ------------------ | ---------------------------------- |
| Children per node | Any number     | Max 2 (left/right) | Max 2 (left/right)                 |
| Data ordering     | No rule        | No rule            | Left < Node < Right                |
| Efficiency        | Varies         | Varies             | Efficient for search (if balanced) |
| Use case          | Hierarchy, DOM | Base structure     | Fast search / sort operations      |

  PROPERTIES THAT WILL HELP IN SOME QUESTIONS:
  1. Perfect binary tree if height = h, then total nodes = 2^(n+1) - 1;
  2. Leaf Nodes in perfect Binary Tree  = 2^h.

  3. if we have N number of levels then how many minimum levels we have = log(N+1) levels atleast.
    => if we have N number of nodes the minimum possible height is log(N+1) minimum levels.
  4. If we have strict binary tree, N leaf nodes then internal nodes = N - 1;
    => and Number of Leaf Nodes = Number of internal Nodes + 1;
  5. Number of Leaf Nodes = 1 + Number of internal Nodes with 2 children.(not includeing root node).

      WHEN TO USE: when we want to delete a root node post order traversal will first delete left tree and right tree and then root this makes sure nodes is deleted and memory safe(or no memory leaks).

  => The upheap operation (also called bubble-up or heapify-up) is used in a heap (usually a binary heap) when you insert a new element

    => we use ArrayList to make a priority Queue insted of insted of linkedlist because it takes O(n) where arraylist only takes O(log n).
    => What is Upheap?
     1. In a min-heap, the parent node must be less than or equal to its children.
     2. When you insert a new element, it is added at the end of the heap (i.e., the last position in the array).
     3. You then compare it with its parent:
      A. If the new element is smaller than its parent, you swap them.
      B. Repeat this process until the heap property is restored.

HASHING: if the returned hashcode is large here hashing is added to hashcode and reduce it to smaller number to store in the hash table. because we cannot

so, for retriving data the data will be stored by generating a hash function and if the value is large, hashing is used to reduce the size of the generated hashcode.
and if two keys has same hasing value generated that is called collision.
=> for collision we have two ways to deal with it.
1. chaining. 
2. open addressing.

=> CHAINING: (chaining the elements like linked list inside the index to form all the elements)
    => If collision we make an assumption that every key equally likely to be hashed to any slot in the table independent of all the keys that are hashed.
    => we call this as Simple uniform hashing. where, n = number of keys in table and m = size of table.
        load factor (alpha) = n/m; (expected key per slot).

    HASH FUNCITONS:
    1. DIVISION METHOD: h(k) = k % m; where m is size of array. sometimes they take prime number that is not too close to the power of 2 or 10;
    2. Multiplication Method: h(k) = [(a*k)%2^w] >> (w-r);

    UNIVERSAL HASHING: h(k) = [(a*k+b) % p] % m;  where a & b are random numbers (0,1,2,...p-1) where p is a large prime number.
    SIZE OF TABLE: m = tita(n); all the time. where if m is small the table is slow and if table is big we are wasteing resources.
    IDEA OF THE TABLE: start small and then grow big.
    SIZE: 
    ADDING=> When we add +1, for every rotation to double the size, the time it takes is n = o(1+2+3+...+n) = O(n^2);
    MULTIPLY => when we multiply *2 (double the size) => m*=2;, the time it takes is n = o(1+2+4+8...n) = O(N).
    => on an average if the add 1 item it takes o(1) amotized constant time.
    => if the values is half the size thatis => n = m/2; we can shrink the size by half => m/2;
    => when n = m/4 then half the size that is 0(1) amotized time.

OPEN ADDRESSING: (one item per slot => m >= n)
