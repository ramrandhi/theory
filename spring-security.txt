SPRING SECURITY: 

WHAT IS SECURITY:
1. Privacy Protection : protecting user data.
2. Trust : application should be trust worthy with security.
3. Integrity: How it is working.
4. Compliance: there are some digital world laws that need to be complied by our application.


1. AUTHENTICATION - Authenticaion is proving who you are.
2. AUTHORIZATION  -  Authorization is about what you're allowed to do after you've proven who you are.

kEY SECURITY PRINCIPLES :
1. Least Privilige - to perform certain task users should have minimum permissions to do that task.
2. Secure by Design - security should be managed when the design is being created.
3. Fail safe defaults - without requireing additional configurations they should be secure by default.
4. secure Communication - whatever data that is incoming or outgoing that should be secured.
5. input validation - external apis need to be validated from threats like sql injection etc.
6. Auditing and Logging - need to have logging or auding module to check who is doing what...

ABOVE DIAGRAM SECURITY FLOW: 
1. When the request comes from client it goes throw some number of spring security filters for it validation by intercepting the
    username and password and create autentication object from those credentials.
2. That Authenticaion object is handed over to Authenticaion Manager which will use the object to Authenticaion Provider like OAuth2
    to verify the autentication object and needs two password encoder and UserDetailsService.
3. UserDetailsService will be called with certain methods like loadByUserName() to load the user data like what ever roles he has in
    the system and convert that object into user detail object and that object is sent back to Authenticaion provider for validation
    of username and password if they are correct then authentication provider created a complete object by populating more data from
    the fetched object like roles etc.
4. That complete object is returned back to autentication manager and it returns back to authentication filters then the security
    context is set. (Security context is the context that will be available through out the request and it stores information about the
    the authentication is stored).
5. Security context will take care of the incoming requests that are coming from that client in that logged in time.

JWT: (JSON WEB TOKENS)

without jwt:
1. no advanced features like expiration time.
2. can be decoded easily.
3. should we go for "custom token system".


THEN WITH JWT: 
JSON web tokens are an open, industry standard.

HOW JSON WEB TOKEN IS GENERATED:

 Client                         Server
  |                               |
  | 1. User enters credentials    |
  |-----------------------------> |
  |                               |
  | 2. Server authenticates       |
  |    and generates token        |
  |                               |
  | 3. Token issued to user       |
  |<----------------------------- |
  |                               |
  | 4. Token sent in API request  |
  |-----------------------------> |
  |                               |
  | 5. Server validates token     |
  |                               |
  | 6. If valid, authorize        |
  |<----------------------------- |
  |    Else, return error         |

1. jwt token contains 3 parts. 1. headers, 2. payload like properties of token, 3. secret pass key.

We require 3 files or classes to implement JWT Authentication.
1. JwtUtils
    -> Contains utility methods for generating, parsing and validating JST's.
    -> Include generating a token from a username, validating a JWT and extracting the username from a token.
2. AuthTokenFilter
    -> Filters incoming requests to check for a valid Jwt in the header, setting the authentication context if the token is valid.
    -> Extracts JWT from request header, validates it and configures the spring security context with user details if the token is valid.
3. AuthEntryPointJwt
    -> Provides custom handling for unauthorized requests, typically when authentication is required but not supplied or valid.
    -> When an unauthorized request is detected, it logs the error and returns a JSON response with an error message, status code,
       and the path attempted.
THIS CLASS IS FOR IMPLEMENTINH SECURITY LOGIC AND CREATING BEAN OF SPRING SECURITY.
4. SecurityConfig:
    -> Configures spring security filters and rules for the application.
    -> Sets up the security filter chain, permitting or denying access based on paths and roles. it also configures session management
       to stateless, which is crucial for JWT usage.
