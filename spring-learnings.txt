1). SPRING IOC (INVERSION OF CONTROL) CONTAINER :

The Spring IoC container is responsible for managing the beans and their dependencies in a Spring Boot application. The IoC container includes:

Sping boot container types :

a). Bean Factory - 
	1. bean factory is the simplest container in spring where it provides the basic functionality like dependency injection and managing beans.
	2. Bean factory lazy loads beans (only when they are requested and resolves the dependencies.
	3. bean Factory is primarily used for small scale applications where all the features of application context is not required.
	4. Example: XmlBeanFactory (deprecated) and programmatic configuration via BeanFactory.

b). Application Context - (Application context is the extension of bean factory that adds more enterprise level features.
	1. Application context eagerly loads all the beans at startup, unlike BeanFactory.
	2. Supports event propagation, declarative mechanisms for creating beans, internationalization (I18N), and AOP (Aspect-Oriented Programming).
	3. Offers additional functionality such as bean post-processing, lifecycle management, and integration with Spring’s web and transaction services.
	
	*. Types of application context - 
	1. AnnotationConfigApplicationContext - loads the context using annotations (Typically used in spring boot).
	2. ClassPathXMLApplicationContext - loads the context from XML Configuration file.
	3. WebApplicationContext - A specialized ApplicationContext for web applications, integrated with Spring's MVC framework.

CIRCUIT BREAKERS : 

	A circuit breaker is a design pattern commonly used in distributed systems to enhance the resilience and fault tolerance of applications. It acts as a safeguard to prevent an application from repeatedly trying to perform an operation that is likely to fail, thus avoiding unnecessary strain on system resources.

	How it Works:
	The circuit breaker monitors the interactions between services and tracks failures. It has three main states:
	
	Closed:
	In this state, the circuit breaker allows requests to flow through to the service.
	It monitors the success and failure rates.
	If failures exceed a predefined threshold within a specified time, the circuit breaker trips to the Open state.
	
	Open:
	In this state, the circuit breaker blocks all requests to the service.
	This prevents overloading the failing service and allows it time to recover.
	After a certain time, the circuit breaker transitions to the Half-Open state to test the service.
	
	Half-Open:
	In this state, the circuit breaker allows a limited number of test requests to the service.
	If these requests succeed, the circuit breaker transitions back to Closed.
	If they fail, it goes back to the Open state.
	Benefits of Circuit Breakers:
	Improved Resilience: Prevents cascading failures in distributed systems.
	Resource Optimization: Reduces load on failing services and frees resources for other tasks.
	Faster Recovery: Allows failing services time to recover before resuming normal operation.
	Better User Experience: Provides fallback responses rather than letting requests hang indefinitely.
	Real-World Analogy:
	A circuit breaker in an electrical circuit cuts off power to prevent overload and damage. Similarly, a software circuit breaker "trips" to protect a system from continuous failed interactions.

	Implementation in Spring Cloud:
	Spring Cloud provides built-in support for circuit breakers via libraries like:

	Resilience4j: A lightweight and modular framework for circuit breakers and other fault tolerance patterns.

EXCEPTION HANDLING IN SPRING BOOT : Spring internally has SpringErrorHandler class that can handle the minor exceptions but for specific exception that need to be implemented we need to use @ControllerAdvice to framework level for spring to intercept all the exceptions thrown at controller level.

-> @ControllerAdvice -> Handles Exceptions from both RestController and Controller
-> @ControllerAdvice(annotations = RestController.class) -> Handles Exceptions only comes from RestControllers.
-> @ControllerAdvice(annotations = Controller.class) -> Handles Exceptions only from Controllers
-> @Order(Ordered.HIGHEST_PRECEDENCE) -> this gives the higest priority to this current class annotated with @ControllerAdvice to handle i mean high priority.
-> @ExceptionHandler(Exception.class) -> this is a method level annotation used to specify the type of exception that method should be handling

1. It is a specialized annotation used to handle exceptions thrown by controllers.
2. It works globally unless scoped to specific packages or controllers.
3. You can define multiple exception handlers for different exception types.

@Order(Ordered.HIGHEST_PRECEDENCE)
@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(IllegalArgumentException.class)
    public ResponseEntity<String> handleIllegalArgumentException(IllegalArgumentException ex) {
        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(ex.getMessage());
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<String> handleAllExceptions(Exception ex) {
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body("An unexpected error occurred");
    }
}

Different end points and its usage
1. /health -> shows application health information, useful for checking the status of the application such as database connectivity, disk space and custom health checks.
2. /info -> Displays arbitrary application information, commonly used to display application version, git commit information etc.
3. /metrics -> the show the information of the performance and behaviour of the running application.
4. /loggers -> allows you to query and modify the logging level of your application's loggers (bascially used to change the log in levels of the user at run time)
5. /beans -> provides a complete list of all the spring beans in your application
6. /shutdown -> Allows your application to be gracefully shut down.
