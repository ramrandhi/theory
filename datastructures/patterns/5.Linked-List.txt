In-Place Reversal Pattern
  1.  The In-Place Reversal technique is key for manipulating linked lists efficiently without using extra space. 
      It’s especially useful when you need to:
  2.  Reverse an entire or partial list
  3.  Detect or reorder nodes
  4.  Process elements in reversed segments

  Pattern Recognition

| Pattern                   | Use When                                                        |
| ------------------------- | --------------------------------------------------------------- |
| Full list reversal        | You need to reverse the entire list                             |
| Partial/sublist reversal  | You need to reverse a portion (e.g., between positions m and n) |
| Group reversal            | You need to reverse nodes in k-groups or alternate groups       |
| Frequency/structure check | Reversal helps validate structure (e.g., palindrome, reorder)   |

🧠 Bonus Concepts to Practice

| Concept          | Description                                                  |
| ---------------- | ------------------------------------------------------------ |
| 3-pointer method | `prev`, `curr`, `next` — for iterative in-place reversal     |
| Dummy node       | Simplifies head edge cases in partial or group reversal      |
| Midpoint logic   | Used in problems like Palindrome and Reorder List            |
| Frequency maps   | Used in problems like Palindrome to validate value structure |
