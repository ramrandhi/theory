Top K Elements (Heaps) Pattern

Heaps (priority queues) are ideal for efficiently retrieving the top K largest/smallest elements or maintaining dynamic 
collections sorted by priority.

This pattern is especially useful when you need to:

    Find K largest or smallest elements
    Track running medians or frequency-based top elements
    Merge sorted sequences efficiently

🧠 Pattern Recognition

| Pattern                     | Use When                                      |
| --------------------------- | --------------------------------------------- |
| K largest/smallest elements | Find top K largest or smallest efficiently    |
| Running median              | Maintain median dynamically with two heaps    |
| Merge K sorted lists/arrays | Merge multiple sorted inputs efficiently      |
| Frequency-based top K       | Find top K frequent elements using a min-heap |

🧠 Bonus Concepts to Practice

| Concept              | Description                                                     |
| -------------------- | --------------------------------------------------------------- |
| Min-heap vs Max-heap | Min-heap for smallest elements, Max-heap for largest elements   |
| Custom comparators   | Use when ordering depends on multiple keys (freq, lex order)    |
| Balancing heaps      | For running median, keep heaps balanced by size                 |
| Lazy removal         | For sliding window median, remove out-of-window elements lazily |
