🔄 Matrix Traversal Pattern

Matrix traversal involves visiting each element of a 2D grid systematically. It’s foundational for many problems involving grids, images, or board games. Variations include traversal by rows, columns, diagonals, spirals, or using search techniques like BFS/DFS.

Common use cases:

    Exploring neighbors
    Searching connected components or islands
    Manipulating matrix in specific orders (spiral, diagonal)

🧠 Pattern Recognition

| Pattern                          | Use When                                     |
| -------------------------------- | -------------------------------------------- |
| Row-wise / column-wise traversal | You need to visit every element sequentially |
| Spiral traversal                 | Traverse matrix in spiral order              |
| Diagonal traversal               | Visit elements along diagonals               |
| BFS/DFS on matrix                | Explore connected cells or shortest paths    |
| Boundary traversal               | Visit only edges or boundaries               |

🧠 Bonus Concepts to Practice

| Concept                | Description                                                                         |
| ---------------------- | ----------------------------------------------------------------------------------- |
| Direction arrays       | Use arrays like `int[] dr = {-1, 0, 1, 0}; int[] dc = {0, 1, 0, -1};` for neighbors |
| Boundary checks        | Always check indices within matrix bounds before accessing                          |
| Visited matrix         | Keep track of visited cells for BFS/DFS                                             |
| Recursion vs Iteration | Use recursion for DFS/backtracking, queues for BFS                                  |
