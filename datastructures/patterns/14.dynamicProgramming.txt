🧩 Dynamic Programming (DP) Patterns

Dynamic Programming is used to solve problems that can be broken down into overlapping subproblems with optimal substructure. 
It helps avoid redundant calculations by storing intermediate results.

Common scenarios:

    Counting combinations or ways
    Optimizing values (max/min)
    Subsequence or substring problems

🧠 Pattern Recognition

| Pattern              | Use When                                                   |
| -------------------- | ---------------------------------------------------------- |
| 1D DP (Linear)       | Problems on sequences or arrays with subproblems in 1D     |
| 2D DP                | Problems on grids, strings, or two sequences               |
| Knapsack             | Select items with weights and values to optimize objective |
| Subset/Sum partition | Partition sets or find subsets meeting certain criteria    |
| DP with Bitmasking   | Track subsets or states efficiently with bit operations    |
| DP on Trees          | Solve tree-based problems using postorder traversal        |

🧠 Bonus Concepts to Practice

| Concept                 | Description                                   |
| ----------------------- | --------------------------------------------- |
| Memoization             | Top-down caching of recursive calls           |
| Tabulation              | Bottom-up iterative DP tables                 |
| State Compression       | Use bitmasks to reduce DP state space         |
| Overlapping Subproblems | Reuse previously computed results             |
| Transition Formulas     | Carefully define relation between subproblems |
