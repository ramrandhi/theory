What is the Sliding Window Technique?
=> Sliding window is used when you're dealing with:
  1. Arrays or strings
  2. Contiguous subarrays or substrings
  3. You want to calculate something like max, min, sum, count, or frequency within a range.

  Instead of checking all subarrays (O(N²)), you slide a window over the input, adjusting start and end pointers for O(N) performance.

  When to Use Sliding Window:

  | Clue in the Problem                                | Example Problem                           |
| -------------------------------------------------- | ----------------------------------------- |
| Substring/subarray of fixed size                   | Max sum of subarray of size k             |
| Longest/shortest subarray/substring with condition | Longest substring with K distinct chars   |
| First/maximum/minimum element in a window          | Sliding window maximum                    |
| All anagrams of a pattern in a string              | Find all anagrams of "abc" in string      |
| Character/number frequency in a substring          | Longest substring with at least K repeats |
| Count of substrings satisfying condition           | Substrings with at most K distinct chars  |

Types of Sliding Window Patterns:

1. Fixed-size Sliding Window
=> Used when the window size k is constant.
=> Example:

// Max sum of subarray of size k
int maxSum(int[] nums, int k) {
    int max = 0, sum = 0;
    for (int i = 0; i < nums.length; i++) {
        sum += nums[i];
        if (i >= k) sum -= nums[i - k];
        if (i >= k - 1) max = Math.max(max, sum);
    }
    return max;
}

2. Variable-size Sliding Window.
=> Used when you need to expand and shrink the window dynamically based on a condition.
=>  Example:

// Longest substring with at most K distinct characters
int longestSubstringKDistinct(String s, int k) {
    Map<Character, Integer> map = new HashMap<>();
    int start = 0, maxLen = 0;
    for (int end = 0; end < s.length(); end++) {
        map.put(s.charAt(end), map.getOrDefault(s.charAt(end), 0) + 1);
        while (map.size() > k) {
            map.put(s.charAt(start), map.get(s.charAt(start)) - 1);
            if (map.get(s.charAt(start)) == 0) map.remove(s.charAt(start));
            start++;
        }
        maxLen = Math.max(maxLen, end - start + 1);
    }
    return maxLen;
}


3. Two Pointer Variant
=> Sliding window is just a special case of two pointers — often when dealing with sorted arrays, longest/shortest subsequence/subarray.

4. Monotonic Queue + Sliding Window
Used when you need max or min in each window, especially in problems like Sliding Window Maximum.

Checklist: How to Recognize a Sliding Window Problem
✔ Array or string
✔ Substring/subarray
✔ Continuous segment
✔ Looking for:
1. Maximum/minimum
2. Count
3. Sum
4. Frequency

Validity of a condition within a range
✔ Window shrinks or expands based on a condition or limit
✔ The brute force is O(N²), and you can avoid re-computation by storing intermediate results.


Common Problems Solved by Sliding Window:

| Problem                                              | Type               |
| ---------------------------------------------------- | ------------------ |
| Maximum sum subarray of size k                       | Fixed size         |
| Longest substring without repeating characters       | Variable size      |
| Longest substring with at most K distinct characters | Variable size      |
| Find all anagrams of a pattern                       | Frequency tracking |
| Longest subarray with sum <= K                       | Variable + sum     |
| Minimum window substring                             | Shrinking window   |
| Sliding window maximum                               | Monotonic queue    |

Tips:
1. Use a map or frequency array to track counts when dealing with characters or numbers.
2. For variable-size, condition is key: expand end while valid, shrink start when invalid.
3. Dry run on paper helps immensely in understanding shrinking/expanding logic.
