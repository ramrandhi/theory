ALL GIT COMMANDS : 

git version control systems are of 3 types
  1. local version constrol systems            -> In this case the repository is within the system and if system crashes or sharing code not efficient
  2. centralized version control systems     -> In this case there is only one centralized server and if the server fails its gone and no history available
  3. distributed version control system -> here we have local repository and when we want to change to others we can do that easily
  
  to check the user name and email    -> git config --global --list
  to change the username             -> git config --global user.name ""
  to change the email                 -> git config --global email ""
  
| Scenario                         | Command                     |
| -------------------------------- | --------------------------- |
| Unstage before commit            | `git restore --staged file` |
| Undo last commit, keep staged    | `git reset --soft HEAD~1`   |
| Undo last commit, unstage        | `git reset HEAD~1`          |
| Undo last commit, delete changes | `git reset --hard HEAD~1`   |

  
  git init          -> to initilize a project
  git init -b main   -> to initialize the project with branch main insted of default branch name master
  
  git status         -> to check the status
  git logs           -> to check the commits in the current directory
  git commit -a -m "message"     -> here -a is used to directly commit with out staging the changes
  git diff            -> it shows the changes made to the file(works when the files are not staged)
  git diff --staged    -> it shows the changes in terminal but only the files that are staged
  git rm --cached filename           -> to remove a file from repository and history
  git tag -> to know the internal version of the repository 
  => git tagging is of two types 1. annotated tagging, 2. lightweight tagging
  -> annotated tagging is what bascially used as it gives as information about the specific user and specific user information for his commits
  
  git tag -a v1.0 -m "first annotated tagging created"
  git show v1.0  -> gives the discription about the tagging v1.0
  
  git checkout -b feature1 or git checkout --branch feature1  -> to create a new branch with name feature1
  git switch -c feature1 or git switch --create feature1  -> to create a new branch with name feature1
  
  git switch feature1 -> to switch to feature1 branch
  git checkout feature  -> to switch to feature1 branch
  
  git branch -> to see all local branches
  git branch --all  -> to see all the local and remote branches
  
  git branch -d branchName   -> to delete a branch
  git branch --delete branchName  -> to delete a branch
  
  
  SHORT CUT -> git switch -     -> to move from current branch to previous branch
  
  merge the above branch(feature1) to current or old branch(main / default) branch -> git merge feature1 (to do this we need to be in main branch)


+----------------------------------+-------------------------------------------------------------+
| Command                          | Description                                                 |
+----------------------------------+-------------------------------------------------------------+
| git init                         | Initializes a new Git repository                           |
| git clone <repo_url>              | Clones an existing repository                             |
| git status                        | Shows the working directory status                        |
| git add <file>                    | Stages a file for commit                                  |
| git add .                         | Stages all modified and new files                         |
| git commit -m "message"           | Commits staged changes with a message                     |
| git commit --amend                | Amends the last commit                                    |
| git push                          | Pushes changes to a remote repository                     |
| git push -u origin <branch>       | Pushes a branch and sets it as upstream                   |
| git pull                          | Fetches and merges changes from a remote repository       |
| git fetch                         | Downloads objects and refs from another repository        |
| git branch                        | Lists branches                                            |
| git branch <name>                 | Creates a new branch                                      |
| git branch -d <branch>            | Deletes a branch locally                                  |
| git checkout <branch>             | Switches to a different branch                           |
| git checkout -b <branch>          | Creates and switches to a new branch                     |
| git merge <branch>                | Merges a branch into the current branch                   |
| git rebase <branch>               | Reapplies commits on top of another base branch          |
| git log                           | Shows commit history                                      |
| git log --oneline                 | Shows a condensed commit history                         |
| git diff                          | Shows changes between commits, branches, or working dir  |
| git diff <branch>                 | Shows changes between current branch and another branch  |
| git reset <file>                  | Unstages a file                                           |
| git reset --hard <commit>         | Resets repository to a specific commit                   |
| git reset --soft <commit>         | Moves HEAD to commit but keeps changes staged            |
| git revert <commit>               | Creates a new commit to undo a previous commit           |
| git stash                         | Saves changes temporarily                                |
| git stash pop                     | Restores stashed changes                                 |
| git stash list                    | Lists stashed changes                                    |
| git stash drop                    | Deletes a stash                                          |
| git remote -v                     | Shows remote repositories                               |
| git remote add <name> <url>       | Adds a new remote repository                            |
| git remote remove <name>          | Removes a remote repository                            |
| git tag <name>                    | Creates a new tag                                       |
| git tag                           | Lists all tags                                          |
| git tag -d <tagname>              | Deletes a tag                                           |
| git show <commit>                 | Displays details of a commit                           |
| git blame <file>                  | Shows who modified each line of a file                 |
| git rm <file>                     | Removes a file from the repository                     |
| git config --global user.name "Name" | Sets the global username                               |
| git config --global user.email "Email" | Sets the global email                              |
| git cherry-pick <commit>          | Applies a specific commit to another branch            |
| git show-ref                      | Shows references                                        |
| git reflog                        | Shows a history of HEAD movements                      |
| git bisect start                  | Starts a binary search to find a bad commit            |
| git bisect bad                    | Marks the current commit as bad                        |
| git bisect good                   | Marks the current commit as good                       |
| git submodule add <url> <path>    | Adds a submodule                                        |
| git submodule update --init       | Initializes and updates submodules                     |
| git gc                            | Cleans up and optimizes the repository                 |
| git fsck                          | Verifies the integrity of the repository               |
| git clean -f                      | Removes untracked files                                |
| git archive -o <file.zip> HEAD    | Creates an archive of the repository                   |
| git grep "<pattern>"              | Searches for a pattern in files                        |
