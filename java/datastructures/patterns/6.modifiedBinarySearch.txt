Modified Binary Search Pattern
 => Modified Binary Search is used when you're dealing with a sorted array (or conditions acting like one) but with a 
    twist — rotated arrays, duplicates, missing elements, etc. It's a powerful technique for reducing time complexity from 
    linear to logarithmic (O(log n)).

🧠 Pattern Recognition

| Pattern                       | Use When                                                                |
| ----------------------------- | ----------------------------------------------------------------------- |
| Classic binary search         | You search in a normally sorted array                                   |
| Condition-based search        | You're looking for the **first/last occurrence** or element with a rule |
| Rotated sorted array          | Array is sorted but rotated – find pivot or search element              |
| Infinite array / unknown size | Array is unbounded – expand boundaries, then binary search              |
| Mountain or bitonic array     | Array increases then decreases – find peak, then search halves          |

🧠 Bonus Concepts to Practice

| Concept               | Description                                                            |
| --------------------- | ---------------------------------------------------------------------- |
| First/last occurrence | Adjust `right = mid - 1` or `left = mid + 1` carefully                 |
| Pivot-based search    | In rotated arrays, determine which half is sorted to narrow search     |
| Infinite search       | Expand boundary (`high *= 2`) and then perform regular binary search   |
| Binary search tricks  | Always check boundary conditions to avoid infinite loops or off-by-one |
