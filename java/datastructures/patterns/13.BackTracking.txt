🔄 Backtracking Pattern

Backtracking is a general algorithmic technique for solving problems incrementally, building candidates to the solution, and 
abandoning a candidate (“backtracking”) as soon as it is determined that this candidate cannot possibly lead to a valid solution.

It’s widely used in:

    Permutations and combinations
    Subset problems
    Constraint satisfaction problems (e.g., Sudoku)
    Puzzle solving (e.g., N-Queens)

🧠 Pattern Recognition

| Pattern                     | Use When                                                     |
| --------------------------- | ------------------------------------------------------------ |
| Generating all permutations | You need to explore all possible orderings                   |
| Generating all combinations | You want all subsets or combinations                         |
| Subset sum / Partition      | Explore sums or partitions with pruning                      |
| Constraint satisfaction     | Solutions must satisfy constraints (e.g., Sudoku, crossword) |
| Puzzle solving              | Search space with multiple decisions (e.g., N-Queens)        |

🧠 Bonus Concepts to Practice

| Concept        | Description                                            |
| -------------- | ------------------------------------------------------ |
| Pruning        | Cut branches early when invalid                        |
| Sorting input  | Helps skip duplicates                                  |
| State tracking | Use boolean arrays, sets, or bitmasks to track choices |
| Memoization    | Cache intermediate results to avoid recomputation      |
