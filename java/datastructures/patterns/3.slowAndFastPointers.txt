🧠 Slow & Fast Pointers – What You Learn from These:
    How cycles work: detecting, starting point, length.
    How to use speed difference to find mid-points.
    How to use in number-based problems (Happy Number, Find Duplicate).
    How to solve symmetry problems (palindrome, twin sum).
    How to apply in both linked lists and arrays.

| When to Use                             | How to Think                                |
| --------------------------------------- | ------------------------------------------- |
| Need to find **middle** of a list?      | Use `slow` and `fast` (1x, 2x)              |
| Need to **detect cycle**?               | Use slow/fast and look for meeting point    |
| Need to **find cycle start**?           | After meeting, reset one pointer to head    |
| Need to **measure cycle length**?       | Count steps until pointers meet again       |
| Need to process **symmetrical halves**? | Find middle, reverse second half            |
| Need to skip `n` ahead?                 | Offset fast by `n` steps before moving slow |
| Working with **arrays like lists**?     | Use values as next-pointers (Floyd’s idea)  |

🧾 Bonus Tips:
    Always consider using a dummy node in linked list problems to simplify edge cases.
    In array problems, convert index-value relationship into linked-list style jumps.
    For palindrome or twin sum problems, always reverse second half after finding mid.
    For slow/fast pointer loop detection, always ask:
    “Is there a chance of revisiting the same place through consistent steps?”
