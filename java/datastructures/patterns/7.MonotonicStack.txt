📚 Monotonic Stack Pattern
A monotonic stack maintains elements in a specific increasing or decreasing order, and is typically used to:

    Find next/previous greater/smaller elements
    Solve range-related problems
    Handle histogram or span-style challenges

Monotonic stacks are often used when:

    You need to process elements left-to-right or right-to-left
    You're interested in "first larger/smaller before/after"

🧠 Pattern Recognition

| Pattern                 | Use When                                                                     |
| ----------------------- | ---------------------------------------------------------------------------- |
| Next Greater Element    | Find the first greater element to the right or left                          |
| Next Smaller Element    | Find the first smaller element to the right or left                          |
| Increasing stack        | Used for **Next Smaller** or **Previous Smaller** (monotonically increasing) |
| Decreasing stack        | Used for **Next Greater** or **Previous Greater** (monotonically decreasing) |
| Histogram area / ranges | Keep index stack and calculate area when popping                             |

🧠 Bonus Concepts to Practice

| Concept                 | Description                                                                                 |
| ----------------------- | ------------------------------------------------------------------------------------------- |
| Stack holds indices     | Especially in range/area problems like histograms                                           |
| Clear before push       | Keep stack monotonic by popping elements that break the order                               |
| Simulate circular array | Iterate twice, use modulo to wrap around                                                    |
| Contribution counting   | In subarray min/max problems, each element contributes based on its nearest smaller/greater |
